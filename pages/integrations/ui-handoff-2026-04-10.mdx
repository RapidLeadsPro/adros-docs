# UI Handoff (2026-04-10)

Audience: UI/UX engineers and frontend contributors continuing the Adros dashboard overhaul.

## Backend Endpoints Ready Now

All endpoints below are live on `https://api.adros.ai/api/v1`.

### Chat Agent

- `POST /chat` — send a message and get full response
- `WS /ws/chat` — streaming chat response
- `GET /chat/memories` — memory debug endpoint
- Auth: `Authorization: Bearer <user_mcp_token>`

### Command Center / Dashboard

- `GET /projects` — list projects
- `GET /projects/{id}/tasks` — list tasks with subtasks
- `GET /tasks/{id}/events` — task activity feed
- `POST /audits/trigger` — trigger audit
- `POST /tasks/{id}/approve` — approve task
- `POST /tasks/{id}/reject` — reject task with feedback
- `GET /approvals/pending` — pending approvals queue
- `WS /ws/events/{task_id}?token=<mcp_token>` — real-time event stream

## File Storage Status

R2 bucket is provisioned, but file APIs are not implemented yet.

Planned backend endpoints:

- `POST /files/upload` — upload to R2
- `GET /files/{path}` — signed download URL
- `GET /files/browse?client_id=X` — list files for UI browser

Until these are live, file browser UI should be treated as scaffold/placeholder.
