# Adros

**The marketing operating system that turns real ad accounts and strategy into traceable work.**

Adros has three layers:

1. **Deterministic rails** — tasks, event timelines, approvals, blockers, client-scoped files, and scoped integrations. Work is auditable whether it starts in the dashboard or via MCP.
2. **Managed-agent depth** — when the problem is ambiguous (audits, strategy synthesis, creative direction), Adros routes to a **Managed Agent** (CMO for analysis, CD for creative) that can go deep — not a single-shot tool call.
3. **Operator surfaces** — a full dashboard at [app.adros.ai](https://app.adros.ai) where humans see the same spine the agents use: **Command Center** (task inbox, timeline, approvals), **Strategy Workspace** (staged artifacts and masters), **Files** (R2-backed storage), **Campaigns** (live Google/Meta data), and **Adros CEO** (coordination that routes into tracked work).

## What's inside

### Operator surfaces (dashboard)

| Surface | Route | Purpose |
|---------|-------|---------|
| **Command Center** | `/command-center` | Client-scoped task inbox with live event timeline, approve/reject, blockers, comments → follow-up tasks |
| **Strategy Workspace** | `/workspace` | Staged strategy artifacts (research → economics → structure → copy/creative → audit → deploy), canonical masters, promote, deploy request |
| **Files** | `/files` | Upload and browse backed by **Cloudflare R2**, scoped by user/org/client |
| **Campaigns** | `/campaigns` | Live Google Ads and Meta Ads performance when accounts connected; scope and fallback behavior surfaced |
| **Adros CEO** | `/ceo` | Coordination chat that auto-routes into tracked tasks (audit, creative, deploy) |
| **Connections** | `/accounts` | Google/Meta account linking and scope selection |

### Backend capabilities

- **4,022 performance-tested ad patterns** — creative moat across styles and frameworks
- **Managed Agents** — `adros_cmo` (audits, research, structure, optimization), `adros_cd` (copy/creative), `adros_deployer` (launch/changes), `adros_ceo` (routing/intake)
- **Daily autonomous monitor** — 6 health checks (creative fatigue, search waste, RSA health, conversion health, Advantage+, experiments), HMAC-signed webhook to your URL
- **Quality track** — audit scorecards, guardrails, and artifact persistence on completed audits (gated by `AUDIT_QUALITY_TRACK_ENABLED` env var)
- **Contract layer** — deterministic deliverable specs per lifecycle stage
- **Creative pipeline** — generation anchored to pattern blueprints, product guardrails
- **Memory layer** — business context, personas, weekly logs scoped per client
- **~74 MCP tools** — Streamable HTTP at `/mcp` for Claude, Cursor, or custom agents
- **Artifact canon** — versioned analysis, mastered operating specs, traceable from strategy through deploy

Dashboard and backend features ship continuously. **What you see on `app.adros.ai` / `api.adros.ai` may lag the latest Git `main` by one deploy.** For integration testing, verify against your target environment; treat this site as the **intent** of the system.

## Get started

### MCP (3 minutes)

```json
{
  "mcpServers": {
    "adros": {
      "url": "https://api.adros.ai/mcp",
      "headers": { "X-User-Token": "lyr_YOUR_TOKEN" }
    }
  }
}
```

1. Sign up at [app.adros.ai](https://app.adros.ai) and copy your MCP token from **Settings → MCP**
2. Paste the JSON above into your AI client's MCP config
3. Restart the client — Adros tools load automatically

### Dashboard (no setup)

Sign in at [app.adros.ai](https://app.adros.ai) and use Command Center, Strategy Workspace, Files, Campaigns, and CEO directly. Same backend, same auth model, same tasks and artifacts.
